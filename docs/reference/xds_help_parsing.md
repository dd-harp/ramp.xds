# Parsing

After solving, use `get_` functions to inspect outputs.

The outputs are generated by a set of functions that parse the outputs.
Some important details are:

- Outputs are parsed and stored on the xds object as `xds_obj$outputs`

- Running `xds_solve` again will overwrite any existing outputs.

- for differential equations, the return value from `deSolve` is stored
  as `deout`

- `time` is stored as `xds_obj$time`

- state variables are parsed and attached by name as
  `xds_obj$outputs$orbits.`

  - `XH` holds parsed outputs for the **XH** component

  - `MY` holds parsed outputs for the **MY** component

  - `L` holds parsed outputs for the **L** component

- several dynamical terms are automatically computed and attached

- the last state of the system is stored as `mod$outputs$last_y`

- `get_*_orbits` functions return the orbits for first species, by
  default.
